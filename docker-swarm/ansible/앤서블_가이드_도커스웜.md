## 사전 준비

### LB(Octavia) 설정

> ⚠️ Octavia가 구성되어 있다는 가정하에 진행합니다.

로드밸런서 풀 멤버에 **리더 인스턴스(`swarm-mg`)를 제외한** 아래 두 인스턴스를 추가하여 로드밸런서를 만들어줍니다.

| 인스턴스 | 포트 |
| --- | --- |
| `swarm-mg2` | 80 |
| `swarm-mg3` | 80 |

만들어진 로드밸런서에 Floating IP를 붙여주세요.

<br>

### Ansible 파일 전송 및 압축 해제

GitHub에서 `ansible` 폴더를 다운받아 zip으로 압축한 뒤, `mgmt` 노드로 전송합니다.

**Windows 환경 (cmd / PowerShell):**

```powershell
scp C:\Users\user\Downloads\ansible.zip user1@172.16.8.200:~/
```

**mgmt 노드에서 압축 해제:**

```bash
unzip ansible.zip "ansible/*"
```

압축 해제 후 파일 구조를 확인합니다.

```bash
ls ~/ansible
# ansible.cfg  group_vars  hosts.ini.example  roles  site.yml
```

**권한 설정:**

```bash
cd ~/ansible
chmod 755 .
chmod 644 ansible.cfg
```

## SSH 설정

### mgmt 노드 키 페어 생성

`mgmt` 노드에서 키 페어를 생성하고 공개키를 `network` 노드에 복사합니다.

```bash
ssh-keygen -t rsa -b 4096 -N "" -f ~/.ssh/id_rsa
ssh-copy-id user1@172.16.8.101
```

<br>

### network 노드 sudo 설정

network 노드 접속 시 root가 아닌 일반 사용자라면, **network 노드**에서 아래와 같이 sudo 패스워드 없이 실행되도록 설정합니다.

```bash
# user1인 경우
echo 'user1 ALL=(ALL) NOPASSWD:ALL' | sudo tee /etc/sudoers.d/user1
```

<br>

### qrouter UUID 확인

**network 노드**에서 아래 명령어로 qrouter의 UUID를 확인합니다.

```bash
sudo docker exec neutron_l3_agent ip netns list
```

출력 예시:

```
qrouter-3d41fc31-6dbf-4463-a70b-6f6951e18cab (id: 3)
...
```

`qrouter-...` 형태의 UUID를 이후 `hosts.ini` 설정에서 사용합니다.

## hosts.ini 설정

> ⚠️ GitHub에 올린 `hosts.ini.example`은 구조 참고용입니다. 실제 `hosts.ini` 파일을 새로 만들고 `{{}}` 부분을 수정해주세요.

> ⚠️ `/etc/kolla/globals.yml`에 `neutron_plugin_agent: "openvswitch"`로 설정되어 있는지 확인하세요.

```ini
[swarm_managers]
swarm-mg  ansible_host={{ swarm_mg_ip }}  swarm_node_name={{ instance_name }}
swarm-mg2 ansible_host={{ swarm_mg2_ip }} swarm_node_name={{ instance_name_2 }}
swarm-mg3 ansible_host={{ swarm_mg3_ip }} swarm_node_name={{ instance_name_3 }}

[db]
db ansible_host={{ db_ip }}

[all:vars]
ansible_user=ubuntu
ansible_ssh_private_key_file={{ ssh_key_path }}
ansible_ssh_common_args='-o StrictHostKeyChecking=no -o ProxyCommand="ssh -i {{ bastion_key_path }} -o StrictHostKeyChecking=no -q {{ bastion_user }}@{{ bastion_host }} sudo ip netns exec {{ qrouter_ns }} nc %h %p"'
image_repo=jaeyun1/board
```

**수정이 필요한 항목:**

| 변수 | 설명 |
| --- | --- |
| `{{ swarm_mg_ip }}` / `{{ swarm_mg2_ip }}` / `{{ swarm_mg3_ip }}` | 각 Swarm 매니저 인스턴스의 private IP |
| `{{ instance_name }}` / `{{ instance_name_2 }}` / `{{ instance_name_3 }}` | 각 인스턴스 이름 |
| `{{ db_ip }}` | DB 인스턴스의 IP |
| `{{ ssh_key_path }}` | 인스턴스 접속에 사용할 키 페어 경로 (예: `~/ansible/openstack2.pem`) |
| `{{ bastion_key_path }}` | mgmt 노드에서 network 노드 접속에 사용할 키 경로 (예: `/home/user1/.ssh/id_rsa`) |
| `{{ bastion_user }}` | network 노드의 사용자명 (예: `user1`) |
| `{{ bastion_host }}` | network 노드의 IP (예: `172.16.8.101`) |
| `{{ qrouter_ns }}` | [위에서 확인한 qrouter UUID](#qrouter-uuid-확인) |

> 인스턴스 이미지는 **Ubuntu** 기준입니다. 다른 이미지를 사용하는 경우 `ansible_user`를 함께 변경해주세요.

## Ansible Vault 설정

이 플레이북은 **Ansible Vault**로 민감 정보를 암호화합니다. 복호화를 위해 vault 패스워드 파일을 설정해야 합니다.

```bash
# vault 패스워드 파일 생성 (파일 내용은 notion에 있는 docker swarm의 6/30 clean up 참조)
nano ~/.ansible_vault_pass

# 권한 설정
chmod 600 ~/.ansible_vault_pass
```

`ansible.cfg`에 vault 패스워드 파일 경로를 지정합니다.

```ini
[defaults]
vault_password_file = ~/.ansible_vault_pass
timeout = 60

[privilege_escalation]
become_timeout = 60
```

### 7. 매니저 1대 다운 복구

```bash

```
