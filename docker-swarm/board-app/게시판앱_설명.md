## 소개

간단한 글쓰기/목록/상세 조회 기능을 제공하는 게시판 애플리케이션입니다.
운영 환경에서의 안정성을 고려해 다음과 같은 요소들을 포함하고 있습니다.

- 세션 쿠키 기반의 사용자 식별
- Docker Secrets를 통한 민감 정보(DB 접속 정보) 관리
- Blue / Green 두 가지 테마의 프론트엔드 (무중단 배포 검증용)

<br>

## 주요 기능

| 기능 | 설명 |
| --- | --- |
| 글 작성 | 제목/내용 입력 후 등록 (`Ctrl + Enter` 단축키 지원) |
| 글 목록 | 페이지네이션 지원 (페이지당 20개) |
| 글 상세 | 모달 형태로 상세 내용 표시 |
| 세션 확인 | 현재 세션 ID 및 컨테이너 IP 표시 (배포 검증용) |
| 장애 대응 | DB 커넥션 오류 시 503과 함께 안내 메시지 반환 |

<br>

## 기술 스택

**Backend**
- Node.js 20 (Alpine)
- [Fastify](https://fastify.dev/) 5.x
- `@fastify/cookie`, `@fastify/static`

**Database**
- MySQL / MariaDB (`mysql2`)
- PostgreSQL (`pg`)
- `db_type` Secret 값에 따라 자동 분기

**Infra**
- Docker / Docker Secrets

<br>

## 프로젝트 구조

```
board-app
├── stack_ingress.yml  # LB(ocatvia) 없이 테스트용 (ingress)
├── Dockerfile
├── package.json
├── server.js          # Fastify 서버, 라우트 정의
├── db.js              # DB 커넥션 풀, 쿼리, Failover 로직(실제로는 DB단 MaxScale 사용)
└── public/
    ├── index_blue.html   # Blue 버전 프론트엔드
    └── index_green.html  # Green 버전 프론트엔드
```

> ⚠️ 실제 배포 시에는 `public/index.html` 경로로 매핑되어야 합니다.
> `index_blue.html` / `index_green.html` 중 배포 대상에 맞는 파일을 `index.html`로 지정해주세요.

## 시작하기

### 필수 조건

- Docker 및 Docker Compose (Swarm 환경)
- MySQL/MariaDB 또는 PostgreSQL 인스턴스

### Docker Secrets 설정

이 프로젝트는 환경 변수가 아닌 **Docker Secrets**로 DB 접속 정보를 주입받습니다.
아래 Secret들을 사전에 생성해야 합니다.

| Secret 이름 | 필수 여부 | 설명 |
| --- | :---: | --- |
| `db_type` | ✅ | `mysql` 또는 `postgres` |
| `db_primary_host` | ✅ | Primary DB 호스트 |
| `db_primary_port` | ✅ | Primary DB 포트 |
| `db_user` | ✅ | DB 사용자명 |
| `db_password` | ✅ | DB 비밀번호 |
| `db_name` | ✅ | DB(스키마) 이름 |
| `db_replica_hosts` | ❌ | Read Replica 호스트 목록 (콤마로 구분, 예: `replica1:3306,replica2:3306`) |

예시

```bash
echo "mysql" | docker secret create db_type -
echo "primary.db.internal" | docker secret create db_primary_host -
echo "3306" | docker secret create db_primary_port -
echo "board_user" | docker secret create db_user -
echo "supersecret" | docker secret create db_password -
echo "board_db" | docker secret create db_name -
echo "replica1.db.internal:3306,replica2.db.internal:3306" | docker secret create db_replica_hosts -
```

### 실행 방법

먼저 lock 파일이 생성을 위해 먼저 npm install을 해주세요.

```bash
# 이미지 빌드 & 푸시
# 예시에서는 도커 허브를 사용했지만 다른 것을 사용해도 됩니다.
docker build -t {이미지명}:{태그명} .
docker push {이미지명}:{태그명}

# Swarm 배포 (기본값: jaeyun1/board:blue)
# 환경에 따라 stack_ingress.yml 파일에서 replica 수 조정 필요 & web 컨테이너가 올라갈 노드에 label을 web으로 설정하기
# 원하는 이미지를 지정해서 배포하고 싶다면 BOARD_IMAGE로 따로 지정해주세요.
BOARD_IMAGE={이미지명} docker stack deploy -c stack_ingress.yml board-app
```

로컬 단독 실행(Secrets 없이 테스트할 경우, `/run/secrets/` 경로에 직접 파일 마운트 필요):

```bash
npm ci
node server.js
```

기본 포트는 `3000`이며, `PORT` 환경 변수로 변경 가능합니다.

## API 명세

| Method | Endpoint | 설명 |
| --- | --- | --- |
| `GET` | `/` | 메인 페이지 (세션 쿠키 발급) |
| `GET` | `/api/session` | 세션 ID 및 컨테이너 IP 조회 |
| `GET` | `/api/posts?page=1` | 글 목록 조회 (페이지당 20개) |
| `GET` | `/api/posts/:id` | 글 상세 조회 |
| `POST` | `/api/posts` | 글 작성 (`title`, `content` 필수) |
| `GET` | `/health` | 헬스체크 |

**글 작성 요청 예시**

```bash
curl -X POST http://localhost:3000/api/posts \
  -H "Content-Type: application/json" \
  -d '{"title": "제목입니다", "content": "내용입니다"}'
```

## DockerFile
- node:20-alpine 베이스로 이미지 용량을 최소화했습니다.
- npm ci --omit=dev로 package-lock.json 기준 프로덕션 의존성만 설치합니다. (lock 파일이 생성을 위해 먼저 npm install을 사용하세요.)
- stack.yml의 healthcheck(wget)가 동작하도록 wget을 별도로 설치합니다.
- 보안을 위해 root가 아닌 node 유저로 전환 후 실행합니다 (USER node).
- server.js, db.js, public/ 디렉터리만 이미지에 포함되며, 그 외 파일(stack_ingress.yml, README 등)은 포함되지 않습니다.

## Blue-Green 배포

`public/index_blue.html`, `public/index_green.html` 두 가지 테마 파일을 제공합니다.
배포 시 둘 중 하나를 `index.html`로 바꿔 끼우는 방식으로 무중단 배포(B/G) 검증이 가능합니다.

- 화면 우측 상단의 세션 뱃지(🔑)로 현재 응답 중인 **세션 ID / 컨테이너 IP**를 확인할 수 있어, 배포 전환 시 트래픽이 어느 컨테이너로 라우팅되는지 쉽게 검증할 수 있습니다.
- Blue: 블루 테마 / Green: 그린 테마로 시각적으로 버전을 구분합니다.

<br>
