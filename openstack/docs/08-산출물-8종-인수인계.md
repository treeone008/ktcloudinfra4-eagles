## §8. SSH (mgmt / control)

| 항목 | 상태 |
|------|:----:|
| mgmt `.200` → tenant SSH | ✅ **control ProxyJump** (FIP 직접 라우팅 불가) |
| control → tenant SSH | ✅ |

### mgmt에서 (산출물 공식)

```bash
# user1@mgmt-01, 키: ~/.ssh/id_rsa (control project_key 복사본)
# control 비번: user1

ssh -o IdentitiesOnly=yes -J user1@172.16.8.100 -i ~/.ssh/id_rsa \
  ubuntu@172.16.8.219 hostname          # swarm-mg ✅

ssh -o IdentitiesOnly=yes -J user1@172.16.8.100,ubuntu@172.16.8.219 -i ~/.ssh/id_rsa \
  ubuntu@192.168.101.31 hostname        # db01
```

> mgmt는 provider FIP(`.219`)에 **No route to host** → **mgmt → control → FIP/private** 가 정상 패턴.

### control에서

```bash
ssh -i /root/.ssh/id_rsa ubuntu@172.16.8.219   # swarm-mg
ssh -i /root/.ssh/id_rsa ubuntu@172.16.8.243   # swarm-mg2
ssh -J ubuntu@172.16.8.219 ubuntu@192.168.101.31  # db01 등
```
