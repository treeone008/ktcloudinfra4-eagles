## 2. 이슈 · 해결

| # | 이슈 | 원인 | 해결 | 상태 |
|:-:|------|:-----|:-----|:-----|
| 1 | `.219`·`.243` 전부 unreachable (재부팅 직후) | L3 agent 미기동 | network `.101` neutron_l3 재시작 | ✅ |
| 2 | `.243` FIP만 실패 (L3 OK 후) | public NIC 나중 추가 | `--nic` 2개로 재생성 | ✅ |
| 3 | SSH `HOST IDENTIFICATION CHANGED` | VM 재생성 | `known_hosts` 삭제 | ✅ |
| 4 | mg-01→`.22` ping FAIL, nc 22 OK | ICMP 경로 / 부팅 중 | cloud-init 완료 후 SSH로 검증 | ✅ |

### 교훈 (재발 방지)

| 주제 | 내용 |
|:-----|:-----|
| VM 생성 | **public IP는 생성 시 `--nic`에 포함** (`port create` 후 추가는 cloud-init 미적용 위험) |
| FIP | `openstack server add floating ip … --fixed-ip-address <public-fixed>` 필수 (private 포트 선택 방지) |
| 재부팅 | FIP 전체 실패 → **L3 agent** 먼저 · tenant VM SHUTOFF면 `server start` |
| SSH | VM 재생성 후 **`ssh-keygen -R <FIP>`** |
| External net | FIP 풀 네트워크 이름 **`public1`** (`public` 아님) |

## 5. 재부팅 · FIP 빠른 복구 명령

```bash
source /root/venv/bin/activate && source /etc/kolla/admin-openrc.sh
openstack server list -c Name -c Status
openstack server start swarm-mg-01 swarm-mg-02 db01 db02  # SHUTOFF 시

ssh root@172.16.8.101 "docker restart \$(docker ps --format '{{.Names}}' | grep neutron_l3)"
sleep 25
ping -c2 172.16.8.219 && ping -c2 172.16.8.243
```

## 6. swarm-mg-02 재생성 레시피 (재현용)

```bash
PUB_NET=$(openstack network show project-public-net -f value -c id)
PRIV_NET=$(openstack network show project-private-net -f value -c id)

openstack floating ip unset --port 172.16.8.243
openstack server delete swarm-mg-02
sleep 20

openstack server create --flavor m1.swarm --image ubuntu \
  --nic net-id=$PUB_NET,v4-fixed-ip=192.168.100.22 \
  --nic net-id=$PRIV_NET \
  --security-group project-sg --key-name project_key \
  --server-group swarm-server-group \
  --host compute-node-02 \
  swarm-mg-02

sleep 90   # cloud-init 완료 대기
openstack server add floating ip swarm-mg-02 172.16.8.243 --fixed-ip-address 192.168.100.22
ssh-keygen -f /root/.ssh/known_hosts -R 172.16.8.243
ssh -i /root/.ssh/id_rsa ubuntu@172.16.8.243 "hostname"
```
