## 1. 아키텍처 개요

```
VMware Workstation (호스트)
├─ control          (.100)  Controller + Monitoring + (현재 mgmt 겸함)
├─ network          (.101)  Neutron (L3 / DHCP / OVS / Metadata)
├─ storage          (.102)  Cinder
├─ compute1         (.103)  Compute (nova-compute)
├─ compute-node-02  (.104)  Compute (nova-compute)
├─ compute-node-03  (.106)  Compute (nova-compute) ← 07-04 합류
└─ mgmt-01          (.200)  관리/SSH/Ansible 진입점  ← 제작 중
   VIP / Horizon    (.105)

OpenStack tenant 인스턴스 (10/11, monitor는 VMware .110)
├─ swarm-mg-01  (.100.20, FIP .219) @ compute1
├─ swarm-mg-02  (.100.22, FIP .243) @ compute-node-02
├─ swarm-mg-03  (.100.24) @ compute-node-03
├─ db01 / db-proxy-02 / lb-02 @ compute-node-02
├─ db-proxy-01 / lb-01 @ compute1
└─ db02 / automation-01 @ compute-node-03
```

- **VMware VM = 인프라 그릇** (control/compute/network/storage/mgmt)
- **tenant 인스턴스 = OpenStack 안에서 만든 VM** (swarm-mg, db…)

## 8. 표준 생성 레시피 (재현용 요약)

> 새 환경에 처음부터 세팅하거나 인스턴스를 다시 만들 때. 상세/트러블슈팅은 `06` 가이드.

```bash
# 외부망 + FIP 풀
openstack network create --external --provider-physical-network physnet1 --provider-network-type flat public1
openstack subnet create --network public1 --subnet-range 172.16.8.0/24 --gateway 172.16.8.2 \
  --allocation-pool start=172.16.8.201,end=172.16.8.250 --no-dhcp public-subnet

# 프로젝트망
openstack network create project-public-net
openstack subnet create --network project-public-net --subnet-range 192.168.100.0/24 --gateway 192.168.100.1 --dns-nameserver 8.8.8.8 project-public-subnet
openstack network create project-private-net
openstack subnet create --network project-private-net --subnet-range 192.168.101.0/24 --gateway 192.168.101.1 --dns-nameserver 8.8.8.8 project-private-subnet

# 라우터
openstack router create project-router
openstack router set --external-gateway public1 project-router
openstack router add subnet project-router project-public-subnet
openstack router add subnet project-router project-private-subnet

# 보안그룹 / 키페어 / 플레이버는 §4·§5 참조

# 인스턴스 (anti-affinity 예: db01)
GID=$(openstack server group list -f value -c ID -c Name | awk '/db-server-group/{print $1}')
NET=$(openstack network show project-private-net -f value -c id)
openstack server create --flavor m1.medium --image ubuntu \
  --nic net-id=$NET,v4-fixed-ip=192.168.101.31 \
  --security-group project-sg --key-name project_key --hint group=$GID db01

# swarm-mg-02 (public+private 동시 — port 나중 추가 금지)
PUB=$(openstack network show project-public-net -f value -c id)
PRIV=$(openstack network show project-private-net -f value -c id)
openstack server create --flavor m1.swarm --image ubuntu \
  --nic net-id=$PUB,v4-fixed-ip=192.168.100.22 \
  --nic net-id=$PRIV \
  --security-group project-sg --key-name project_key \
  --server-group swarm-server-group --host compute-node-02 swarm-mg-02
openstack server add floating ip swarm-mg-02 172.16.8.243 --fixed-ip-address 192.168.100.22
```

## 9. 운영/점검 명령 모음

```bash
# 서비스/에이전트 상태
openstack compute service list

# 리소스 전체 조회
openstack image list; openstack flavor list
openstack network list; openstack subnet list; openstack router list
openstack security group list; openstack keypair list
openstack floating ip list; openstack server group list
openstack server list

# 인스턴스 배치/상태
openstack server show <name> -c status -c addresses -c "OS-EXT-SRV-ATTR:host"

# compute에서 실제 VM 확인
docker exec nova_libvirt virsh list --all

# 사설망 인스턴스 핑 (network 노드 namespace 경유)
PRIV=$(openstack network show project-private-net -f value -c id)
ssh root@172.16.8.101 "ip netns exec qdhcp-$PRIV ping -c3 192.168.101.31"
```
