# OpenStack 구성 담당 — 작업 체크리스트 (김현도)

> 팀장 지시: **구성만** 담당 / **검증은 팀장**  
> 원칙: **한 번에 X → 하나씩 생성 → 확인 → (테스트 후) 삭제 가능**  
> 최종: **VM 목록 + 문서화 + 예산 산출**

---

## 0. 작업 원칙

| 원칙 | 설명 |
|------|------|
| 순차 작업 | 네트워크 → 라우터 → SG → 이미지/플레이버 → VM **한 대씩** |
| 확인 후 진행 | 각 단계 CLI/Horizon으로 확인 후 다음 단계 |
| 테스트 삭제 | 연습용 VM은 확인 후 삭제 가능 (문서에 기록) |
| DB 배치 | **다른 Compute 노드** + **Server Group(anti-affinity)** 사용 |
| 문서 동시 작성 | 구성할 때마다 [문서화-마스터표](./문서화-마스터표.md) 갱신 |

## 작업 순서 요약 (한 줄 플로우)

```
이미지·플레이버 확인
  → 네트워크 → 서브넷 → 라우터 → external 연결
  → 보안그룹 + 규칙
  → Floating IP 할당·연결
  → Server Group (DB)
  → VM 1대씩 생성·확인
  → Cinder 볼륨 (테스트 → DB용)
  → mgmt SSH 검증
  → 표·예산 문서 → 팀장 전달
```
