# 프로젝트 로드맵 — 김현도 (OpenStack/Linux/Network)

> 이 문서 = **전체 방향·현재 상태·다음 할 일**의 기준. 작업할 때마다 갱신.

**최종 갱신:** 2026-07-02

---

## 1. 프로젝트가 뭐냐

> **OpenStack(IaaS) 위에 웹 서비스를 자동 배포하고, 모니터링·장애 대응까지 하는 시스템.**

```
OpenStack (IaaS 기반)
  └─ Docker Swarm (Web/WAS 컨테이너)
     + MariaDB(db01/db02) + MaxScale (DB 고가용성)
     + Prometheus/Grafana/Alertmanager (모니터링)
     + Ansible (자동화)
     + (최종) Aodh/Heat/Senlin (Auto Scaling)
```

**목표:** 장애 나도 서비스 유지 + 자동 복구 + (최종)자동 확장 시연.

---

## 2. 팀 역할

| 담당 | 사람 |
|------|------|
| **Linux / Network / OpenStack** | **김현도 (나)** ← IaaS 기반 전체 |
| Ansible | 권건우 |
| Docker Swarm | 박재윤 |
| Database | 김인태 |
| Monitoring | 윤진수 |

→ 나는 **"땅을 까는 사람"**. VM·네트워크·보안·스토리지·배치를 만들어 다른 팀원이 그 위에 올리게 함.

---

## 3. 최종 목표 아키텍처 (VMware 7노드)

```
VMware Workstation Pro
├─ mgmt-01            ← 외부 관리/Ansible/SSH/복구
├─ controller-node-01 ← OpenStack Controller
├─ network-node-01    ← Network
├─ storage-node-01    ← Cinder
├─ compute-node-01    ← swarm-mg-01, db02, db-proxy-01, lb-01
├─ compute-node-02    ← swarm-mg-02, db01, db-proxy-02, lb-02
└─ compute-node-03    ← swarm-mg-03, monitor-01, automation-01
```

- **VMware VM = 7개** (인프라 호스트)
- **OpenStack tenant VM = 11개** (위 compute 안에서 생성)

---

## 4. 내 작업 방식 (팀장 요구)

> VM/설정 **1개씩 만들기 → 테스트 → 압축 → 팀장 전송 → 다음**.  
> 팀장이 시연 PC에서 받아 **최종 배포·검증**. (나는 직접 시연 X)

---

## 5. 핵심 개념 (헷갈리지 말 것)

| 구분 | 의미 |
|------|------|
| **VMware VM** | control/compute/network/storage… = OpenStack 돌리는 **그릇** |
| **tenant VM** | swarm-mg, db… = OpenStack **안에서** 만든 인스턴스 (compute 안에 올라감) |
| tenant 11개 만들기 | VMware VM 11개 X → **compute 안에 11개** 생성 |
| compute 2·3대 필요 이유 | VM 개수 X → **anti-affinity·분산** 때문 |
| tenant VM 띄우려면 | control + **compute + network** (+storage는 Cinder 시만) |
